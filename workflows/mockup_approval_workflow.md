# WF-07 — Mockup Approval Workflow

**Trigger:** Mockup ready for review (luxury/premium projects)  
**Output:** Mockup Approved / Revise / Rejected

---

## MOCKUP HIERARCHY

```
Level 1: Material Sample
   → Consultant approval
   → Physical sample board or color chip

Level 2: Workshop Sample
   → Consultant + Client approval
   fabricated in workshop, full detail

Level 3: Site Mockup
   → Brand rep + Consultant + Client review
   Built on-site in actual conditions
```

## APPROVAL PROCESS

```
1. Contractor notifies: Mockup ready for review
2. Agent loads: quality_tier.md (check tier requirements)
3. Agent verifies: all prior levels approved
   → If prior level NOT approved → HOLD, return to contractor

Level 1 Review:
   - Material matches spec requirement
   - Color/texture/finish acceptable
   - Consultant sign-off obtained

Level 2 Review:
   - Fabrication quality check
   - Detail accuracy vs drawings
   - Consultant + Client sign-off obtained

Level 3 Review (Site Mockup):
   - Installed in actual conditions
   - Raking light check (luxury mandatory)
   - Interface with adjacent materials
   - Brand rep review (if branded project)
   - All parties sign-off obtained
```

## DECISION

```
ALL levels approved → MASS PRODUCTION RELEASE
Any level rejected → REVISE MOCKUP → re-review
Incomplete documentation → HOLD until complete
```

## RULES

1. **No mass production without all three approvals**
2. Agent holds execution start until mockup sign-off logged
3. Mockup photos archived in project record
4. Approved mockup becomes reference standard for all subsequent work
5. Changes after mockup approval = new submittal required
